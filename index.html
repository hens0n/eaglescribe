Setup checklist

Short guide so dictation works. Does not block hotkeys or Settings. Audio and transcripts stay on this machine for default dictation.

Microphone

So we can hear you. Grant mic access for EagleScribe.

System Settings → Privacy & Security → Microphone → enable EagleScribe

Accessibility

So we can paste into other apps (clipboard + simulated paste).

System Settings → Privacy & Security → Accessibility → enable EagleScribe

On some macOS versions, Input Monitoring may also be needed for reliable global shortcuts — not required to finish setup; use window Start/Stop if hotkeys fail.

Whisper model

Local speech-to-text needs a ggml model file. Set path under Settings → Whisper model, then Load. Or run npm run model:download (see README).

We never ask for Screen Recording or Full Disk Access for core dictation.

Microphone

Capture needs mic access (PipeWire / PulseAudio on most desktops). Grant when the OS prompts, or check session audio settings.

Hotkeys & paste

Global shortcuts and paste reliability depend on session type (X11 vs Wayland) and packages. Check $XDG_SESSION_TYPE. Pure Wayland may not register global hotkeys — use in-window Start / Stop / Cancel. On paste failure, text stays on the clipboard for manual paste.

See README · Linux hotkeys & paste and research/linux-hotkey-paste-spec.md.

Whisper model

Set path under Settings → Whisper model, then Load. Or npm run model:download (see README).

Local-first: audio and transcripts stay on this machine for default dictation. No macOS Accessibility or Screen Recording steps apply here.

Hotkeys

Global hotkeys unavailable — use window controls. On Linux the current stack needs X11 for global shortcuts; pure Wayland often cannot register. In-window Start / Stop / Cancel still work.

Command Mode LLM

Select text → hold command hotkey → speak instruction. Localhost only (Ollama / llama-server).

Clipboard

On successful paste, put your prior clipboard text back after a short delay. Off = leave the transcript on the clipboard. Failed paste always keeps the transcript for manual paste.

Speech-to-text

After stop, drop quiet padding at the start and end before Whisper (dictation and Command Mode). Mid-utterance pauses are kept. Off = send the full capture buffer. Applies to the next completed take.

Menu bar

Takes effect the next time EagleScribe starts. When on: no Dock icon; tray remains; main window still shows on launch. Restore via tray left-click or Show Window.

        Ready

        Set aside about 4–6 minutes. Tuning runs entirely on this device and never learns from dictation history.

        • It can only learn corrections covered by the fixed built-in English phrases—not unseen names or jargon.
        • You’ll read every phrase twice, in two separated readings.
        • You explicitly approve every proposed Correction Rule.
        • Approved rules must pass a different required Verification Pass before they become active.
